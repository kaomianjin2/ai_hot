# Coding Rules

## 开发前

- 修改代码前必须读取相关代码，至少包含被修改函数和调用方。
- 明确影响范围：文件、函数、接口、用户可见行为。
- 明确验证方式。
- 修改超过 100 行时，先重新评估是否能拆小。

## 修改边界

- 只修改当前任务直接相关文件。
- 禁止修改无关格式、注释和配置。
- 禁止添加未请求的配置项。
- 禁止新增通用抽象，除非明确复用不少于 2 处。

## 编码风格

- 使用函数优先，除非已有代码要求 class。
- 使用提前返回，减少嵌套层级。
- 变量名必须表达完整语义，禁止 `a`、`b`、`tmp` 这类命名。
- 注释只解释为什么，不描述代码表面行为。
- 删除本次修改引入的未使用变量、函数、导入。

## 禁止

- 未经允许安装或升级依赖。
- 未经允许修改 `.env`、`.gitignore`、密钥、证书、生产配置。
- 未经允许修改数据库结构、接口定义、部署流程。
- 未经允许删除文件。
