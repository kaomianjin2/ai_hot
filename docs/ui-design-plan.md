# AI Hot Radar UI Design Plan

## 1. 结论

UI 方向改为参考 HotPulse 截图的科技风单页控制台，不再使用三栏工作台布局。

本阶段交付物是 `design/ai-hot-radar.html`，用于确认 Web 端视觉、布局和交互方向。

## 2. 参考截图结论

已用 Playwright 打开并截图用户提供的 3 个参考链接，截图位于：

```text
output/playwright/reference-1.png
output/playwright/reference-2.png
output/playwright/reference-3.png
```

从参考图提取的 UI 特征：

- 顶部品牌栏：左侧 logo + 产品名，右侧立即扫描按钮和通知按钮。
- 横向功能 Tab：热点雷达、监控词、搜索。
- 暗色科技背景：深蓝黑底、中心弱光、细网格质感。
- 卡片式统计概览：总热点、今日新增、紧急热点、监控词。
- 主内容单列流：排序筛选工具栏 + 热点卡片列表。
- 监控词页面：大输入框 + 添加按钮 + 两列开关卡片。
- 搜索页面：大搜索框 + 搜索按钮 + 结果卡片。

## 3. 产品定位

AI Hot Radar 是一个 Web 端 AI 热点监控控制台，用于第一时间发现 AI 大模型、AI 编程、AI 工具、开源项目、论文和行业新闻热点。

第一版只做 UI 设计稿，不接真实后端。

桌面端规则：

- 主容器最大宽度 `1176px`。
- 内容居中显示。
- 顶部品牌栏固定在页面上方视觉首屏。
- 统计卡使用 4 列。
- 热点卡使用单列大卡片。
- 监控词卡使用 2 列。

### 移动端

移动端规则：

- 横向 tabs 可滚动。
- 统计卡在窄屏下改为单列。
- 搜索框和按钮上下排列。
- 热点卡片单列展示。
- 不允许横向滚动。
- 所有按钮高度不低于 `44px`。

## 6. 视觉方向

视觉风格为科技风 HotPulse 控制台。

设计特征：

- 深蓝黑背景。
- 中心蓝色弱光。
- 细网格背景纹理。
- 蓝青渐变主按钮。
- 高亮状态使用青蓝色。
- 成功状态使用绿色。
- 紧急状态使用红色。
- 卡片边框使用低对比蓝灰色。
- 卡片圆角不超过 `8px`。

禁用：

- 营销式 hero。
- 大面积紫蓝渐变。
- 装饰性光球。
- 卡片套卡片。
- Bootstrap 蓝。

## 8. 组件规范

### Topbar

- 显示 logo、产品名、中文副标题。
- 右侧显示 `立即扫描` 和通知按钮。
- 通知按钮显示 `9+` 红色角标。

### Tabs

- 横向展示 `热点雷达`、`监控词`、`搜索`。
- 当前 tab 使用半透明高亮卡片。

### Stats Cards

- 4 个核心指标：总热点、今日新增、紧急热点、监控词。
- 数字使用等宽字体。
- 不同指标使用不同状态色。

### Hot Radar

- 标题行显示 `实时热点流` 和自动更新时间。
- 排序 chips 包含：最新发现、最新发布、重要程度、相关性、热度综合。
- 筛选区包含筛选和重置按钮。
- 搜索面板包含输入框和搜索按钮。

### Hot Card

- 展示来源、主题、时间。
- 展示标题。
- 展示摘要。
- 展示标签。
- 右侧展示 `查看` 按钮。

### Monitor Keywords

- 展示关键词输入框和添加按钮。
- 展示关键词开关卡片。
- 卡片中可显示命中热点数量。

## 9. 动效设计

- `button press`: 150ms，缩放到 `0.98`。
- `card hover`: 180ms，上移 `-2px`，边框高亮。
- `focus-visible`: 2px 青色描边。
- `primary button`: 保持蓝青渐变和弱阴影。
